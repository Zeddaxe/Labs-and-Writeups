Hashing and Cryptography

Cracking random hashes:

You find a hash but you do not know what type of hash it is.

Then you can use hash-identifier to identify the hash type by syntax.

cat hash.txt | hash-identifier 

or 

hash-identifier jg236ua687b23gntgr2785tgy5h4

Note:

A more modern and powerful alternative to hash-identifier is a tool called Haiti if you're interested:

https://github.com/noraj/haiti

Then you can crack that hash using :

john --format=raw-md5 --wordlist=rockyou.txt hash.txt

Decode base64 data

cat encodedData.txt | base64 -d > decodedData.txt
or 
base64 -d encodedData.txt > decodedData.txt

Encryption and Decryption Using GPG (Gnu Privacy Guard)

Symmetric (single key)

Encrypt=
gpg --cipher-algo AES-256 --symmetric file.txt

then, 

Decrypt=
gpg file.txt.gpg    or   gpg --decrypt file.txt.gpg

enter the passphrase and you get your file.

Asymmetric (public and private key)

Encrypt a File

To encrypt a file using GPG, you'll need the recipient's public key. If you want to encrypt the file for yourself, you can use your own public key. Here's the command to encrypt a file:

 gpg --encrypt --recipient [recipient_email_or_key_id] [file_to_encrypt]

Replace [recipient_email_or_key_id] with the email address or key ID of the recipient's public key. If encrypting for yourself, use your own email address or key ID. Replace [file_to_encrypt] with the path to the file you want to encrypt.

For example, to encrypt a file named example.txt for yourself:

gpg --encrypt --recipient you@example.com example.txt

This command will create an encrypted version of example.txt with the file extension .gpg.

Decrypt a File

To decrypt an encrypted file using GPG, you'll need the private key associated with the public key used for encryption. Here's the command to decrypt a file:

gpg --decrypt

Replace [encrypted_file.gpg] with the path to the encrypted file you want to decrypt.

For example, to decrypt the encrypted file example.txt.gpg:

gpg --decrypt example.txt.gpg

Follow any prompts to enter your passphrase or provide any necessary authentication for decryption.

Additional Tips:

  • Use -output [output_file] option to specify the output file when encrypting or decrypting to avoid overwriting the original file.
  • Ensure you keep your private key secure and do not share it with others.
  • Share your public key with trusted parties for them to encrypt files that only you can decrypt.

But, what if you do not have the passphrase? Then, as a hacker, you can have an option to crack the passphrase using Brute Force with John The Ripper.

Cracking GPG Passphrase

gpg2john file.txt.gpg > hash then, john --wordlist=rockyou.txt --format=gpg hash
